# Fitness v1.0.2 — Contract-Driven Pass 1 Walker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the irony gap: let `/ux-cycle` Phase B drive the fitness engine with a ux-architect component contract instead of hand-rolling its own `DazzleAgent` dispatch. Adds a new `walk_contract` Pass 1 mission that translates a contract's quality gates into ledger EXPECT/ACTION/OBSERVE triples, plumbs a `component_contract_path` kwarg through `run_fitness_strategy` and `_build_engine` into `FitnessEngine`, and rewrites the `/ux-cycle` Phase B snippet to call the strategy.

**Architecture:** A new `walk_contract` async function in `src/dazzle/fitness/missions/contract_walk.py` mirrors the shape of the existing `walk_story` walker — it is deterministic, records to the same `_LedgerLike` interface, returns a `WalkResult`, but operates on a `ComponentContract` (parsed by the existing `parse_component_contract`) rather than on a DSL story. Each quality gate becomes one ledger step: `expect = gate.description`, `action_desc = "observe contract gate"`, `observed_ui = await observer.snapshot()`. Actual observation is deferred to an injected `_ObserverLike` protocol so unit tests can pass an in-memory stub and the real strategy can wrap a Playwright page. `FitnessEngine.__init__` grows two optional kwargs (`contract_paths`, `contract_observer`) and Pass 1 iterates contract walks after story walks. `run_fitness_strategy` accepts `component_contract_path: Path | None = None`, and a new inline `_ContractObserver` class adapts the existing `_PlaywrightBundle.page` to the observer protocol. No navigation is performed in v1.0.2 — the contract walker observes whatever page is loaded by the time Pass 1 fires (defaults to about:blank for fresh bundles). URL inference from contracts is a v1.0.3 concern.

**Tech Stack:** Python 3.12+, `dataclasses`, `typing.Protocol`, `asyncio`, existing `ComponentContract` + `parse_component_contract` from `dazzle.agent.missions._shared`, existing `WalkResult` shape from `dazzle.fitness.walker`, existing `_LedgerLike` interface.

**Context for the engineer:** The existing Pass 1 walker is `src/dazzle/fitness/walker.py::walk_story(story, executor, ledger) -> WalkResult`. It iterates `story.steps` (each a dict with `action`, `url`/`selector`/`value`, `expect`) and records each step against a sync ledger via `record_intent(step, expect, action_desc)` and `observe_step(step, observed_ui)`. `walk_contract` mirrors this shape exactly but operates on a `ComponentContract` parsed from a ux-architect markdown file. A `ComponentContract` (defined at `src/dazzle/agent/missions/_shared.py:367`) has `component_name: str`, `quality_gates: list[QualityGate]`, `anatomy: list[str]`, `primitives: list[str]`. Each `QualityGate` has `id: str` (snake_case) and `description: str`. The walker does not drive actions — contract gates are *behavioural assertions*, not action sequences. So each gate becomes one ledger step: expect = gate description, action_desc = a fixed string like `"observe contract gate"`, and observed_ui = the result of calling `await observer.snapshot()`. A new `_ObserverLike` protocol is defined locally in this module — it has just one method, `async snapshot() -> str`. Unit tests pass a fake observer that returns canned strings.

```python
"""Contract-driven Pass 1 walker (v1.0.2 task 1).

Mirrors the shape of ``walker.walk_story`` but drives the ledger from a
parsed ux-architect ``ComponentContract`` instead of a DSL story. Each
quality gate becomes one ledger step: its description is recorded as the
EXPECT, the walker calls an injected observer's ``snapshot()`` method to
capture the current UI state as OBSERVE, and a fixed action description
marks the step as a contract observation rather than a user action.

The walker is deterministic — no LLM calls. The observer is injected so
unit tests can pass an in-memory stub and the fitness strategy can wrap
a Playwright ``page`` with a thin adapter.
"""

from __future__ import annotations

from typing import Protocol

from dazzle.agent.missions._shared import ComponentContract
from dazzle.fitness.walker import WalkResult


class _ObserverLike(Protocol):
    async def snapshot(self) -> str: ...


class _LedgerLike(Protocol):
    def record_intent(self, step: int, expect: str, action_desc: str) -> None: ...
    def observe_step(self, step: int, observed_ui: str) -> None: ...


_ACTION_DESC = "observe contract gate"


async def walk_contract(
    contract: ComponentContract,
    observer: _ObserverLike,
    ledger: _LedgerLike,
) -> WalkResult:
    """Drive the ledger through one observation per quality gate.

    For each gate in ``contract.quality_gates``:

    1. Record the intent with ``expect = gate.description``
    2. Call ``await observer.snapshot()`` to capture the UI state
    3. Record the observation with ``observed_ui = <snapshot>``

    Exceptions raised by ``observer.snapshot()`` are captured into the
    result's ``errors`` list and a synthetic ``"error: <msg>"`` observation
    is recorded, so the walk always produces symmetric intent/observation
    counts per step.
    """
    result = WalkResult(
        story_id=f"contract:{contract.component_name}",
        persona="fitness_contract",
        steps_executed=0,
    )

    for idx, gate in enumerate(contract.quality_gates, start=1):
        ledger.record_intent(step=idx, expect=gate.description, action_desc=_ACTION_DESC)
        result.steps_executed += 1

        try:
            observed = await observer.snapshot()
        except Exception as e:  # noqa: BLE001 — surfacing the error into the ledger is the point
            observed = f"error: {e}"
            result.errors.append(f"step {idx}: {e}")

        ledger.observe_step(step=idx, observed_ui=observed)

    return result
```
