# PEP 695 Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the 18 deferred `UP040/UP046/UP047` sites to PEP 695 native type-parameter syntax (`class C[T]`, `def fn[T]()`, `type X = …`) and remove the ruff ignores, at the current `>=3.12` floor — no floor change, no support dropped.

**Architecture:** Behaviour-preserving syntax modernization. ruff `--unsafe-fixes` does the mechanical conversion per file; a manual review removes the now-dead module-level `TypeVar` declarations and checks bounds / the `type`-alias laziness; mypy 2.1 (strict) + the existing 17,887-test suite are the safety net. One file per task, ignores removed last.

**Tech Stack:** Python 3.12+ (PEP 695), ruff (UP040/UP046/UP047), mypy 2.1, pytest. Spec: `docs/superpowers/specs/2026-06-09-pep695-adoption-design.md`.

**Conventions for every task:**
- Run all tooling via `PYENV_VERSION=dazzle-dev <cmd>` (the repo `.python-version` is `3.14`, which pyenv can't select; this override gives the 3.12 dev venv with all deps). `uv run --python 3.12 <cmd>` also works.
- The per-file ruff check uses an explicit `--select`, which overrides the still-present ignores, so it verifies *that file* is fully converted before the ignores are removed globally in the final task.
- "Dead TypeVar": after a file's classes/functions are converted, its module-level `X = TypeVar("X")` lines are unused — delete them. If mypy/tests then fail, a non-converted consumer still needs it; restore it and leave that consumer on the classic TypeVar.
