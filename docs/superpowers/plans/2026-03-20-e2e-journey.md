# E2E Journey Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add persona-driven E2E journey testing that walks each persona through their workspaces and stories on a live deployment, producing structured JSONL transcripts, cross-persona pattern analysis, and an HTML report.

**Architecture:** Two-phase mission — Phase 1 (deterministic workspace exploration, no LLM) + Phase 2 (LLM-assisted story verification via DazzleAgent). Data models first, then session writer, then browser automation, then analyser and report, then CLI/MCP integration.

**Tech Stack:** Python 3.12, Pydantic v2, Playwright (sync API), Jinja2, pytest. Anthropic SDK for Phase 2 only.

**Spec:** `docs/superpowers/specs/2026-03-20-e2e-journey-design.md`

### Task 3: Cross-Persona Pattern Analyser

Deterministic analysis of completed journey sessions. No LLM, no browser.

**Files:**
- Create: `src/dazzle/agent/journey_analyser.py`
- Test: `tests/unit/test_journey_analyser.py`

- [ ] **Step 1: Write tests**

Tests should cover:
- **Scope leak detection**: persona A sees populated list for entity X, persona B sees empty — produces CPP with "scope filtering" title
- **FK UUID detection**: steps with observation containing "UUID" or "text input" for ref fields — produces pattern
- **Navigation gap**: entity declared in workspace but step for it has BLOCKED verdict
- **Common failures**: same entity fails for 3+ personas — marked as systemic
- **Dead-end extraction**: steps with DEAD_END verdict → `DeadEnd` objects
- **Nav break extraction**: steps with NAV_BREAK verdict → `NavBreak` objects
- **RBAC button violations**: Edit/Delete buttons observed by personas denied by permit rules → pattern
- **State machine dead-ends**: story verification hits inaccessible state transitions → pattern
- **Empty sessions**: no patterns produced for empty input
- **Single persona**: no cross-persona patterns (need 2+ for comparison)
- **Recommendation generation**: patterns produce prioritized recommendations

Build test data using `JourneyStep` and `JourneySession` model constructors from Task 1.

- [ ] **Step 2: Run tests to verify they fail**

- [ ] **Step 3: Implement analyser**

```python
def analyse_sessions(
    sessions: list[JourneySession],
    appspec: AppSpec | None = None,  # for workspace/entity metadata
) -> AnalysisReport:
```

Pattern detection passes:
1. **Scope filtering**: for each entity, compare row counts across personas. If one persona sees rows and another sees zero → scope issue.
2. **FK UUID**: scan step observations for keywords indicating UUID text inputs.
3. **Navigation gaps**: steps with BLOCKED verdict where the target is a workspace region entity.
4. **Common failures**: group FAIL/BLOCKED steps by entity. If 3+ personas fail on the same entity → systemic.
5. **Dead-ends**: extract steps with DEAD_END verdict into `DeadEnd` objects.
6. **Nav breaks**: extract steps with NAV_BREAK verdict into `NavBreak` objects.
7. **RBAC button violations**: scan observations for Edit/Delete buttons on detail pages for personas that should be denied by permit rules (cross-reference with AppSpec entity access if provided).
8. **State machine dead-ends**: scan verify-phase steps for failed state transitions.
9. **Recommendations**: sort patterns by severity, assign priorities, estimate effort.

Each pattern gets a sequential ID (CPP-001, CPP-002, etc.).

- [ ] **Step 2: Implement `journey` subcommand**

Add to `e2e_app`:

```python
@e2e_app.command("journey")
def journey_command(
    url: str = typer.Option(..., "--url", help="Base URL of the deployment"),
    personas: str = typer.Option("all", "--personas", help="Comma-separated persona names or 'all'"),
    phase: str = typer.Option("all", "--phase", help="'all', 'explore', or 'verify'"),
    output_dir: str = typer.Option("", "--output-dir", help="Override output directory"),
    headless: bool = typer.Option(True, "--headless/--no-headless"),
) -> None:
```

Orchestration:
1. Preflight checks: Playwright installed? Credential file exists? Deployment reachable?
2. Load AppSpec, credentials, resolve persona filter
3. For each persona:
   a. Create Playwright browser context
   b. Phase 1: `run_phase1_exploration()`
   c. Phase 2 (if phase != "explore"): `run_phase2_verification()`
   d. Close browser context
4. Run `analyse_sessions()` → write `analysis.json`
5. Run `render_report()` → write `report.html`
6. Print summary to terminal

- [ ] **Step 2: Add error handling per spec**

Implement all edge cases from Section 5:
- Missing credential file → abort with message
- Playwright not installed → abort with message
- Deployment unreachable → abort with message
- Login failure → record as failed, continue to next persona
- `--phase verify` without prior explore data → skip with warning
