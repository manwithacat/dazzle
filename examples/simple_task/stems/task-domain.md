# Stem: Task domain (simple_task)

## Claim

This example is a **minimal** Dazzle app: a single **Task** entity with CRUD
surfaces and a workspace. Its purpose is pedagogical — entity basics, surfaces,
attention signals — not a product domain model.

## Reconstruct

- Prefer teaching clarity over realistic multi-entity graphs.
- DSL under `dsl/` is the source of truth; don’t invent parallel Python domain.
- SPEC / SPECIFICATION are expressions of this intent, not competing stems.

## Not this

- Expanding into full project management without promoting to another example.
- Copying this app’s domain stems into production apps without rewriting.

- `dsl/`, `SPEC.md`, `SPECIFICATION.md`, `README.md`
