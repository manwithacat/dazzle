## Reading the baseline (hand-written analysis)

**Protocol.** 86 blind images (72 Dazzle fleet screens — 12 example apps × 3
screens × light/dark, above-fold 1440×900, frame-normalized; 14 dialect
references — shadcn examples, Vercel, Linear, Stripe, same frame). 3
independent judges (claude-sonnet-4-6 via Claude Code subagents — the
`dazzle qa taste-panel` API path was blocked on API credits; same blind pool,
rubric, and aggregation code), noise measured by 2 repeat passes over 4
images. 1465 validated scores, 0 collection problems. Apps were **unseeded**
— screens show real empty states; that is part of today's truth, and the
same protocol must be used at re-judge time.

**Headline.** Parity FAILS on all six dimensions. The fleet sits at 2.4–3.2
against references at 5.1–5.8 on a 10-point scale — roughly half the judged
quality, everywhere. This is not one weak dimension; it is a uniform deficit.

**Three widest gaps.**
1. `perceived_craft` — gap **3.24** (2.41 vs 5.64), and the tightest margin
   (0.50): judges agree with each other most about this one. The fleet reads
   as "unstyled scaffolding / template output" (anchor 2) — nothing looks
   loved: default-ish controls, inconsistent radii/borders, no icon language.
2. `spatial_rhythm` — gap **2.75** (2.94 vs 5.69): off-scale gaps, crowding
   beside dead space, weak alignment lines. Empty states amplify this but do
   not explain it (see robustness below).
3. `color_discipline` — gap **2.59** (3.22 vs 5.81): the single blue accent
   exists but semantics are muddy; neutrals don't carry structure.

**Two worst apps.** `invoice_ops` (1.64 pooled) and `hr_records` (1.89) —
both data-dense domains where the missing table/badge/density language hurts
most. Best: `domain_join_co` (3.70) and `ticket_classifier` (3.68). Even the
best app sits ~1.9 points under the reference pool mean (5.54).

**Sampling artifact (known, bounded).** The deterministic trim (sort by
workspace) over-selected `_platform_admin` workspaces under non-admin
personas — RBAC-empty screens (worst screen: 1.27). A secondary reading
excluding that workspace (8 screens) moves means by at most +0.24 and DOWN
for craft/rhythm — the gap is robust to the artifact. Next capture round
should sample persona-matched workspaces instead; the admin-persona
`_platform_admin` screens are actually the fleet's best (5.5–5.8), which
says the bones are capable when a screen has dense, well-matched content.

**Judge noise / locked margins.** Pooled repeat-SD per dimension →
margin = max(0.5, 2×SD): color_discipline 1.63, dark_mode_integrity 1.15,
typographic_hierarchy 1.15, spatial_rhythm 0.82, state_completeness 0.58,
perceived_craft 0.50. All under the 1.5-SD sanity ceiling except
color_discipline (SD 0.815) — acceptable, but future runs should raise the
noise subset above 4 images to tighten it. **These margins are the locked
parity gate for Phases 2–4.**

**What this feeds (Phase 1 rules).** The evidence ranks the levers:
craft-signals first (icons, control styling, radii/border/shadow
consistency), spatial scale discipline second, color/neutral ramp third.
Dark mode is the *smallest* gap (1.97) — mostly because references
themselves score modestly in dark — but Dazzle dark reads as inverted-light
(3.13).
